UX Microcopy

Small copy, high stakes. Error states, form validation, and system messages that reduce friction at critical user moments.

01
Error Message Rewrite
Date Range Validation — Fintech Platform
Microcopy Fintech Error State Form UX
Problem

Engineers were writing all UI copy directly — resulting in system-logic language that made sense to developers but created confusion for users. Error messages used technical terms, negative framing, and no actionable guidance. I identified this pattern across the product, rewrote the copy from the user's perspective, and formalised the standards into the design system so the whole team could apply them consistently going forward.

❌ Engineer-written (Before)
迄日不可以小於起日
States a system constraint in developer logic. Uses field variable names (迄日/起日) that are hard to understand at first glance — not what users see on screen. No guidance on how to fix it.
✓ UX rewrite (After)
請確認填寫的起始日小於結束日
Guides the user with 「請確認」. Uses UI label names (起始日/結束日) that match what's on screen. Actionable and human.
Systemic impact: This wasn't a one-off fix. I audited error states across the product, rewrote them with consistent principles, and documented the guidelines in the design system — giving engineers and designers a shared reference so future copy would be written to the same standard without needing UX writer sign-off on every string.

Terminology Decision — Task Status Labels

During a design review for a task management feature, I identified that two status tags — 未處理 and 待處理 — were being used with overlapping meaning, creating ambiguity for users trying to distinguish between task states.

✓ Three distinct states, zero ambiguity
❌ Problem
未處理 ＆ 待處理
Two tags with near-identical meaning used simultaneously — semantically too close to help users distinguish task states clearly.
✓ Recommendation

Retain one term. 未處理 was chosen for its stronger urgency signal and clearer semantic meaning — directly communicating that action is required.
Why it matters: In task management, status labels directly affect how users prioritise their work. When two labels carry the same meaning, users lose trust in the system. This was a verbal recommendation made during design review — showing that content decisions happen in collaboration, not just in a doc.

02
i18n Localisation
Traditional Chinese to English — Product String Localisation
Localisation i18n Vue.js Cross-functional Fintech
Context

The product needed to support both Traditional Chinese and English — requiring all hardcoded UI strings to be extracted and replaced with i18n variables in Vue. Rather than simply handing a translation spreadsheet to the development team, I worked directly in the localisation process: owning all naming and terminology decisions while the developer handled the technical implementation.

My Contribution
I was responsible for determining how each Traditional Chinese UI string should be expressed in English — ensuring the translated copy was not just accurate, but contextually appropriate for a fintech product audience. This included UI labels, status messages, navigation items, and action buttons.
How We Worked Together
The developer replaced hardcoded strings with Vue i18n variables and managed the technical structure. I provided the terminology decisions and reviewed the output in context — catching cases where a literal translation didn't fit the UI space or user expectation.
Why this matters: Localisation is more than translation — it requires understanding how words behave in UI constraints, how tone shifts across languages, and how terminology choices affect the entire product consistently. Working directly alongside engineering meant fewer handoff errors and faster iteration.

Selected Work — 04

Campaign Copywriting

Strategic narratives built around customer insight, conversion behaviour, and brand positioning.

04
Service Campaign
Interior Design Studio — 讓利放量 Lead Generation Strategy
Copywriting B2C Service Strategy Two Concepts
Context & Brief

An interior design studio needed to attract new clients and convert consultation enquiries into booked appointments. To frame the strategic direction, I referenced Helena Rubinstein's Taiwan re-entry campaign — where anniversary offers and bundled incentives successfully lowered the barrier for first-time buyers. This real-world precedent informed the core concept: 讓利放量 (cut prices to boost volume) — giving without cheapening the brand.

Strategic Reference — 讓利放量
In competitive markets, price incentives can more than double inbound volume while variable costs remain manageable — as seen in promotions like Starbucks' buy-one-get-one events. The insight: use partial cost absorption strategically to remove hesitation, not to signal desperation.
Concept A — Value Framing (Budget-Conscious Homeowners)
「我們透過裝修費用做部分讓利，讓業主可以沒有壓力的諮詢。」
The studio absorbs part of the renovation cost — reframing consultation as risk-free. Targets homeowners who want quality but feel budget pressure. Lowers the mental barrier to making the first enquiry.
Concept B — Efficiency Framing (Busy Professionals)
「六週完工保證 — 不止好，而且又快又好。」
Targets working homeowners whose primary pain point is time, not cost. The 6-week completion guarantee converts uncertainty into a concrete promise — and helps the studio manage production capacity more efficiently.
Strategic thinking: The two concepts address two different customer anxieties — cost and time — using real behavioural insight rather than generic messaging. The foreword grounds both concepts in market evidence, showing the business case behind the creative direction.
